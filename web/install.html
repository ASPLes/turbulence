Installing Turbulence

Turbulence dependencies

Turbulence have two required dependencies:

  1. Vortex Library: which provides the BEEP engine (located at: http://vortex.aspl.es).

  2. Axl Library: which provides all XML services (located at: http://xml.aspl.es).

The following optional dependencies are required to built Turbulence. If not provided some features will be disabled.

  1. Libpcre3: a library that provides perl expressions support. It is used by all expressions used by the turbulence configuration file. If not provided all expressions provided will be matched as is.

    This regular expression library was written by Philip Hazel, and copyrighted by the University of Cambridge, England.

    The software is available at: ftp://ftp.csx.cam.ac.uk/pub/software/programming/pcre/

  2. OpenSSL: a library that provides TLS support. If not provided the BEEP TLS profile won't be available. This dependency is required by Vortex Library not by Turbulence.

    The software is available at: http://www.openssl.org

  3. GNU SASL: a library that provides SASL support. If not provided the BEEP SASL family of profiles won't be available.

    The software is available at: ftp://alpha.gnu.org/pub/gnu/gsasl

In many cases, GNU SASL, Libpcre3 and OpenSSL are already available on your platform. Check your distribution documentation.

Building Turbulence from source on POSIX / Unix environments

It is assumed you have a platform that have autoconf, libtool, and the rest of GNU tools to build software.

Get the source code at the download page: download Turbulence

Turbulence is compiled following the standard autoconf-compatible procedure:


   >> tar xzvf turbulence-X.X.X-bXXXX.gXXXX.tar.gz
   >> cd turbulence-X.X.X-bXXXX.gXXXX
   >> ./configure
   >> make
   >> make install
